## Coordinate System
- Origin (0, 0) is at the **bottom-left corner of the Garage**
- X-axis: horizontal movement (right is positive)
- Y-axis: vertical movement (up is positive)
- All dimensions in meters
- Conversion: 1m = 100 pixels (or 50 for finer detail)

## Technical Requirements

### Node Hierarchy
[b]Root Structure:[/b]
[code]
FloorPlan (Node2D root)
├── WallsContainer (Node2D)
│   ├── Garage_Walls (Node2D)
│   │   ├── LeftWall_CollisionShape2D
│   │   ├── TopWall_CollisionShape2D
│   │   ├── BottomWall_CollisionShape2D
│   │   └── Visual ColorRects
│   ├── Suite_de_sejour_Walls (Node2D)
│   │   ├── ...
│   │   └── (no right wall)
│   └── ... (other room wall groups)
├── FloorContainer (Node2D)
│   ├── Garage_Floor (StaticBody2D)
│   │   └── CollisionShape2D
│   ├── Cuisine_Floor (StaticBody2D)
│   │   └── CollisionShape2D
│   └── ... (all floor regions)
[/code]

### Ownership Rules (CRITICAL)
[codeblock]
All dynamically created nodes must have their owner set to the same owner 
as the context node. Use node.owner if it is not null; otherwise, use node 
itself as the owner. This ensures nodes are persisted correctly in the 
scene file.

The owner must be set AFTER the node is added to the scene tree (via 
add_child()) to avoid "Invalid owner" errors.
[/codeblock]

### Cleanup
When removing existing nodes before rebuilding:
1. Use [code]queue_free()[/code] on nodes to remove
2. Wait or check from scene root to ensure proper cleanup
3. Then create new nodes

## Wall Construction

### Wall Types & Thickness
- [color=green]GREEN walls[/color]: Load-bearing (0.2m = 20cm thick)
- [color=red]RED walls[/color]: Thin separators (0.1m = 10cm thick)
- [color=orange]ORANGE walls[/color]: External insulated (0.5m = 50cm thick)

### Geometry Requirements
- Each wall must have:
  - [b]Collision geometry[/b]: [code]CollisionShape2D[/code] with [code]RectangleShape2D[/code]
  - [b]Visual geometry[/b]: [code]ColorRect[/code] or [code]Sprite2D[/code] for display

## Floor Regions

### Implementation
1. Create [code]StaticBody2D[/code] for each room
2. Add [code]CollisionShape2D[/code] with [code]RectangleShape2D[/code]
3. Ensure rectangles don't overlap
4. Name each floor region: [code]{RoomName}_Floor[/code]

### Visual Representation
- Floor regions: [code]Color(0.9, 0.9, 0.9)[/code]
- Wall colors:
  - Green: [code]Color(0, 0.8, 0)[/code]
  - Red: [code]Color(0.8, 0, 0)[/code]
  - Orange: [code]Color(1, 0.6, 0.2)[/code]

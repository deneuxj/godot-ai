# AIChat Node Design

## REQ-CHAT-0001: Provide a new node type: `AIChat`

### `ai_chat.gd`

Extends `Node`. Registered as a custom node type.

```gdscript
class_name AIChat extends Node

# Properties
@export_multiline var system_prompt: String = ""
@export var api_endpoint: String = ""
@export var api_key: String = ""
@export var model: String = ""

# State
var chat_history: Array[Dictionary] = []

# Signals
signal chat_started()
signal progress(chunks: Array[String])
signal chat_finished(full_response: String)
signal chat_error(error_message: String)
signal context_length_updated(tokens: int, characters: int)
signal status_updated(status: String)
```

## API and Behavior

### `send_message(prompt: String, attachments: Array[String] = [])`
1. Appends a new message to `chat_history`. 
    - If `attachments` is empty, content is a simple `String` (prompt).
    - If `attachments` is NOT empty, content is an `Array` of dictionaries (multi-modal format), including the text prompt and base64-encoded image data for each supported resource path.
2. Emits `chat_started`.
3. **Routing (Optional):**
    - If `use_router` is `true` AND `model` is empty:
        - Emit `status_updated("Processing")`.
        - Call `AIClient.chat` using the `router_model` with a specific system prompt to categorize the request.
        - Result is either `analyst` or `technician`.
        - Emit `status_updated("Thinking" if analyst else "Implementing")`.
        - Select the corresponding model from project settings.
    - Else: Use the node's `model` property or the default project `model`.
4. Prepares messages for the API:
    - If `system_prompt` is not empty, it's added as the first message.
    - Appends `chat_history`.
5. Uses `AIClient` to send a streaming request.
6. Relays `AIClient.progress` signal to `AIChat.progress`.
7. Upon completion:
    - Appends the full response to `chat_history`: `{"role": "assistant", "content": full_response}`.
    - Emits `chat_finished(full_response)`.
8. Handles errors and emits `chat_error`.
9. After any change to `chat_history`, calls internal `_update_context_length()` and emits `context_length_updated`.

### `get_context_length() -> Dictionary`
Returns a dictionary `{"tokens": int, "characters": int}`. 
- Tokens are estimated (e.g., chars / 4) if no official tokenizer is available.
- Characters are the total length of all message contents.

### `compress_context()` (REQ-CHAT-0013, REQ-CHAT-0014)
Triggered when `get_context_length().tokens` exceeds `context_limit`.
1. **Categorize Messages:**
    - `Fixed`: System prompt, Tool definitions.
    - `Vital`: Task specification (first 1-2 user messages), last 5 messages.
    - `Prunable`: Tool outputs > 10 turns old, error correction cycles where a later turn succeeded.
2. **Pruning Strategy:**
    - Iteratively remove `Prunable` items starting from the oldest.
    - If still over limit, truncate `Vital` messages (except task spec and last 2) by removing middle turns.
    - Never remove `Fixed` messages.
3. **Critical Overflow Check:**
    - If after compression the length still exceeds `context_limit`:
        - Return `false` (compression failed to reach safety).
        - Logic in `send_message()` will then abort and emit `chat_error`.
4. **Signal:** Emits `context_compressed` when pruning occurs.

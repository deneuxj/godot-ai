# AgentAssisted3D Node Design

## REQ-NODE3D-0001: Provide a new 3D node type: `AgentAssisted3D`

### `agent_assisted_3d.gd`

Extends `Node3D`. Registered as a custom node type via the plugin entry point (`ai_assistant.gd`).

```gdscript
class_name AIAgentAssisted3D extends Node3D

enum GenerationMode { SCRIPTED_SCENE, NODE_SCRIPT }

# Properties (exposed in editor)
var prompt: String = ""
var texture_attachments: Array[Texture2D] = []
var generation_mode: GenerationMode = GenerationMode.SCRIPTED_SCENE
var generated_node_name: String = "GeneratedNode"
var generated_script: Script = null # Populated in NODE_SCRIPT mode
var last_error: String = "" # Readonly property exposing validation errors

enum GenerationStatus { IDLE, GENERATING, SUCCESS, ERROR }
var generation_status: GenerationStatus = GenerationStatus.IDLE
var status_message: String = ""
var generated_code: String = "" # Last generated content (GDScript or TSCN)

# Signals
signal progress(chunks: Array[String])
signal code_updated(code: String)
```

## REQ-NODE3D-0004: Generation Flow with Error Correction Loop

### Generation Flow

The plugin no longer executes AI-generated code. It saves the AI's response as a file and either instantiates the results of a scripted scene or attaches a generated node script.

```gdscript
func generate():
    var max_retries = _get_project_setting("max_retries", 5)
    generation_status = GenerationStatus.GENERATING
    status_message = "Generating..."
    
    # 1. Build initial prompt with mode-specific instructions
    var messages = PromptBuilder.build(prompt, texture_attachments, generation_mode)
    
    for attempt in range(max_retries):
        # 2. Call AI (streaming)
        var content = await _call_ai(messages)
        
        # 3. Update preview (Extract clean code from markdown if needed)
        var code = ScriptExecutor.extract_code(content)
        generated_code = code
        code_updated.emit(code)
        
        if generation_status == GenerationStatus.IDLE: return # Interrupted
        
        # 4. Validate output (Enhanced check)
        var validation_result = ScriptExecutor.validate_output(code, generation_mode)

        if validation_result.error == null:
            last_error = ""
            # 5. Apply results
            _save_to_disk(content, generation_mode)
            _apply_result(generation_mode)
            generation_status = GenerationStatus.SUCCESS
            status_message = "Generation complete"
            return

        # 6. Error Correction
        last_error = validation_result.error
        status_message = "Fixing error on attempt %d/%d: %s" % [attempt + 1, max_retries, validation_result.error]
        messages = PromptBuilder.build_error_correction(messages, validation_result.error, content)
        ```

        ### Validation Strategy (ScriptExecutor)

        Validation is performed by the `ScriptExecutor` using a multi-layered approach:
        1. **Custom Logger**: A `Logger` instance is registered with the Godot engine to intercept internal parse/load errors during validation.
        2. **GDScript Validation**: Uses the Godot Language Server (LSP) for deep semantic and type checking.
        3. **TSCN Validation**: Removed in favor of Scripted Scene generation.

    generation_status = GenerationStatus.ERROR
    status_message = "Generation failed after %d attempts" % max_retries
```

## REQ-NODE3D-0011: Generation Modes

### Scripted Scene Mode
AI is instructed to output a GDScript that constructs a node hierarchy.
1. The AI generates a script extending `Object` or `RefCounted` with a `build() -> Node3D` method.
2. The plugin executes this script. Errors during construction (missing nodes, invalid properties) are caught and reported to the AI for correction.
3. On success, the returned `Node3D` hierarchy is saved as a `.tscn` file using `PackedScene.pack()`.
*Note: This mode provides superior diagnostics for LLM error correction.*

### Node Script Mode
AI is instructed to output a valid Godot `.gd` script extending `Node3D` (or the appropriate type).
The resulting file is saved to `res://generated/<node_name>.gd`.
The `AgentAssisted3D` node attaches this script to itself: `set_script(load(path))`.

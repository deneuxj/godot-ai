# AgentAssisted3D Node Design

## REQ-NODE3D-0001: Provide a new 3D node type: `AgentAssisted3D`

### `agent_assisted_3d.gd`

Extends `Node3D`. Registered as a custom node type via the plugin entry point (`ai_assistant.gd`).

```gdscript
class_name AgentAssisted3D extends Node3D

# Properties (exposed in editor)
var prompt: String = "":
    set(val):
        prompt = val
        if _prompt_changed():
            _on_prompt_changed()
    get: return prompt

var texture_attachments: Array[Texture2D] = []

enum GenerationStatus { IDLE, GENERATING, SUCCESS, ERROR }
var generation_status: GenerationStatus = GenerationStatus.IDLE
var status_message: String = ""

# API overrides (null = use project settings)
var api_endpoint: String = "":
    set(val): api_endpoint = val
    get: return val if val != "" else _get_project_setting("base_url")
var api_key: String = "":
    set(val): api_key = val
    get: return val if val != "" else _get_project_setting("api_key")
var model: String = "":
    set(val): model = val
    get: return val if val != "" else _get_project_setting("model")
```

### Plugin Registration

In `ai_assistant.gd`, the node is registered with Godot's classDB:

```gdscript
class_name AIAssistantPlugin extends EditorPlugin

func _enter_tree():
    _register_node_type()

func _register_node_type():
    var script = preload("res://ai_assistant_plugin/agent_assisted_3d.gd")
    ClassDB.register_class("AgentAssisted3D")
    # Register as subclass of Node3D for editor visibility
```

The node appears in the "Add Node" dialog under the plugin's category.

## REQ-NODE3D-0004: When added to a scene, the node shall use AI to process the prompt and generate a subtree

### Lifecycle

```gdscript
func _ready():
    if prompt != "" and generation_status == GenerationStatus.IDLE:
        # Check for cached generation first
        if not _has_valid_cache():
            generate()
```

### Generation Flow with Error Correction Loop

```gdscript
const MAX_RETRIES := 5

func generate():
    generation_status = GenerationStatus.GENERATING
    status_message = "Generating..."
    emit_signal("generation_started")
    
    # 1. Build initial prompt (design/0002-aiintg.md)
    var messages = PromptBuilder.build(prompt, texture_attachments)
    var script_text := ""
    var success := false
    
    # 2. Attempt generation with error correction loop
    for attempt in range(MAX_RETRIES):
        # 2a. Call AI (design/0002-aiintg.md)
        script_text = await AIClient.chat(messages)
        status_message = "Generating... (attempt %d/%d)" % [attempt + 1, MAX_RETRIES]
        
        # 2b. Execute script (design/0005-safety.md)
        var error_result = await ScriptExecutor.execute_with_error(script_text, self)
        
        if error_result.error == null:
            # No errors - success
            success = true
            break
        
        # 2c. Append error to chat history for correction
        status_message = "Fixing error on attempt %d/%d: %s" % [attempt + 1, MAX_RETRIES, error_result.error]
        
        messages = PromptBuilder.build_error_correction(messages, error_result)
    
    if success:
        # 3. Persist (design/0004-persist.md)
        _save_cache(script_text)
        
        generation_status = GenerationStatus.SUCCESS
        status_message = "Generation complete"
        emit_signal("generation_finished")
    else:
        generation_status = GenerationStatus.ERROR
        status_message = "Generation failed after %d attempts" % MAX_RETRIES
        emit_signal("generation_finished")
```

## REQ-NODE3D-0005: The generated hierarchy shall be persisted and reused across editor sessions

See `design/0004-persist.md`. The generated GDScript is saved as a `.gd` resource to `res://generated/`. The child node tree is persisted via Godot's standard scene serialization. On `_ready()`, the node checks for valid cache before regenerating.

## REQ-NODE3D-0006: A new hierarchy shall be generated automatically when the prompt changes

### Prompt Change Detection

```gdscript
var _prompt_hash: String = ""

func _prompt_changed() -> bool:
    var new_hash = MD5.hash_text(prompt)
    if new_hash != _prompt_hash:
        _prompt_hash = new_hash
        return true
    return false

func _on_prompt_changed():
    if generation_status != GenerationStatus.GENERATING:
        _invalidate_cache()
        generate()
```

---

## REQ-NODE3D-0007: The user shall be able to force re-generation with the same prompt via a button in the editor dock

The `generate()` method can be called explicitly. The editor dock includes a "Generate" button (see `design/0003-editor.md`). The method does not check if the prompt has changed — it always triggers a fresh AI call.

```gdscript
# Force regenerate (ignores prompt hash)
func force_generate():
    _invalidate_cache()
    generate()
```
